# Генератор подання про втрату

Інструмент для швидкого формування тексту подання про втрату безпілотного літального апарата (БпЛА). Заповнюєте поля у формі — програма генерує готовий текст, який можна скопіювати одним натисканням.

## Як запустити

Подвійним кліком відкрийте файл **`Report_2.2.html`** у будь-якому браузері (Chrome, Edge, Firefox тощо).

Більше нічого встановлювати не треба:

- працює **офлайн** — база з ~14 600 населених пунктів уже вбудована в сам файл; інтернет потрібен лише як запасний варіант, якщо координата виходить за межі бази;
- це єдиний файл, який потрібен для роботи.

## Як користуватися

1. Заповніть поля форми:
   - **Дата** і **Час** втрати;
   - **Тип завдання** (розвідувальний політ, бойовий виліт тощо);
   - **Координата** у форматі MGRS (напр. `36U YA 12345 67890`) — після введення повної координати програма сама визначить населений пункт та область і підставить їх у відповідне поле;
   - **Населений пункт, область** — зазвичай заповнюється автоматично з координати; за потреби можна виправити вручну;
   - **Причина втрати** (РЕБ, обстріл, ППО тощо);
   - **Тип БпЛА** та **АКБ до БпЛА**;
   - **Підрозділ** і **Військова частина**.
2. Натисніть **«Згенерувати подання»** — унизу з'явиться готовий текст.
3. Натисніть **«Копіювати в буфер»**, щоб скопіювати текст.
4. **«Нове подання»** очищає форму для наступного запису.

## Корисно знати

- **Підрозділ** і **Військова частина**, а також обрана тема оформлення зберігаються у браузері — наступного разу їх не доведеться вводити знову.
- Угорі є перемикач теми: 🌓 авто / ☀️ світла / 🌙 темна.
- Визначення населеного пункту йде від координати: спершу шукається найближчий НП у локальній базі (радіус 25 км), а якщо такого немає — програма звертається до онлайн-сервісу. Якщо найближчий НП далі за 25 км, з'явиться попередження.

## Поради

- Тримайте файл `Report_2.2.html` у надійному місці (наприклад, зробіть копію). Уся програма — в одному файлі.
- Дані зберігаються лише у вашому браузері на цьому комп'ютері. Якщо відкрити файл на іншому пристрої, поля «Підрозділ/В.Ч.» доведеться заповнити заново.
