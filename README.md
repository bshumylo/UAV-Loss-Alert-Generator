# Генератор подання про втрату БпЛА

Інструмент для швидкого формування тексту подання про втрату безпілотного літального апарата (БпЛА). Заповнюєте поля у формі — програма генерує готовий текст, який копіюється одним натисканням.

## Два режими роботи

- **Просто генератор** — потрібен лише файл `Подання.html`. Ви вводите тип, номер БпЛА та інші поля вручну, отримуєте готовий текст подання. Цього достатньо для більшості випадків.
- **Генератор + реєстр обліку** — додатково підключаєте файл `Реєстр БПЛА.xlsm`. Тоді типи й номери БпЛА підставляються зі списку (лише ті, що ще не втрачені), а одиниця й АКБ — автоматично. У репозиторії лежить готовий **шаблон** реєстру `Реєстр БПЛА (шаблон).xlsm` — з нього й починайте (див. нижче).

## Як запустити

Завантажте останню версію файлу **`Подання.html`** в [Releases](https://github.com/bshumylo/UAV-Loss-Alert-Generator/releases/latest). Відкрийте його подвійним кліком у будь-якому браузері на комп'ютері чи телефоні (Chrome, Edge, Firefox тощо).

Більше нічого встановлювати не треба:

- працює **офлайн** — база з ~14 600 населених пунктів уже вбудована у файл; інтернет потрібен лише як запасний варіант, коли координата виходить за межі бази;
- для генерації подання достатньо цього одного файлу.

## Як користуватися

1. Заповніть поля форми:
   - **Дата** і **Час** втрати;
   - **Тип завдання** (розвідувальний політ, бойовий виліт тощо);
   - **Координата** у форматі MGRS (напр. `36U YA 12345 67890`) — після введення повної координати програма сама визначить населений пункт і область;
   - **Населений пункт, область** — зазвичай заповнюється автоматично; за потреби виправте вручну;
   - **Причина втрати** (РЕБ, обстріл, ППО тощо);
   - **Тип БпЛА**, **Номер БпЛА**, **Одиниця** та **АКБ**;
   - **Учасники** — за бажанням (у сам текст подання не входять, див. нижче);
   - **Підрозділ** і **Військова частина** (у згорнутому блоці).
2. Натисніть **«Згенерувати подання»** — унизу з'явиться готовий текст.
3. Скопіюйте його однією з двох кнопок:
   - **📋 Копіювати для ОЧ** — чистий текст подання (для месенджера, доповіді);
   - **👥 Копіювати для реєстру** — той самий текст плюс рядок «Учасники: …» (для обліку).
4. **«Нове подання»** очищає форму для наступного запису.

## Підключення реєстру (необов'язково)

Якщо ви ведете облік БпЛА у файлі `Реєстр БПЛА.xlsm`, генератор може підтягувати з нього список моделей і **лише тих номерів, що ще не втрачені** — щоб не вводити їх вручну.

1. Розгорніть блок **«Дані з реєстру»** вгорі → **«Підключити реєстр…»** → виберіть свій файл реєстру.
2. Далі поле «Тип БпЛА» показує лише наявні моделі, а «Номер» — випадний список номерів цієї моделі. Одиниця й АКБ підставляються самі.
3. Коли в обліку з'явилася нова техніка — натисніть **«⟳ Оновити»**, і список перечитається з файлу (у Chrome/Edge — без повторного вибору файлу).

Якщо даним понад тиждень, у заголовку блоку з'явиться помаранчеве нагадування оновити список. Файл реєстру читається **локально**, у мережу нічого не передається. Без підключеного реєстру генератор працює як зазвичай — номер вводиться вручну.

## Реєстр обліку: як влаштований шаблон

У репозиторії лежить `Реєстр БПЛА (шаблон).xlsm` — порожня заготовка з такою ж структурою, як робочий файл, але **без реальних даних** (лише кілька прикладних рядків з позначкою «ПРИКЛАД»). Скопіюйте його, перейменуйте на `Реєстр БПЛА.xlsm` і вносьте свою техніку.

Як генератор читає файл:

- **Кожна модель — на окремому аркуші** (`DJI, Autel`, `Лелека`). Перший рядок — заголовки; далі один БпЛА на рядок.
- Значення береться з колонок за назвою заголовка: **`Модель`**, **`Номер`** або **`Борт №`**, **`Од. вим.`**, **`Дата втрати`**. Якщо колонки `Модель` немає — береться назва аркуша.
- **Втрачений БпЛА** = у рядку заповнена колонка **`Дата втрати`**. Такий номер зникає зі списку в генераторі. Тобто щоб «списати» борт — просто впишіть дату його втрати.
- Аркуш **`Довідник`** зіставляє назви моделей з обліку з тим, як вони мають звучати в поданні, і задає одиницю та АКБ за замовчуванням. Колонка **`Показувати у списку` = ні** ховає позицію (АКБ, антени тощо), щоб вона не потрапляла у генератор.
- Аркуш **`Прийом`** із кнопкою/макросом дозволяє вставити згенерований текст подання й автоматично проставити дату втрати потрібному борту (потрібен Excel з увімкненими макросами).

Правила заповнення коротко описані на самому аркуші `Довідник`.

## Корисно знати

- **Підрозділ**, **Військова частина** й обрана тема оформлення зберігаються у браузері — наступного разу вводити не треба.
- Угорі є перемикач теми: 🌓 авто / ☀️ світла / 🌙 темна.
- Населений пункт визначається від координати: спершу найближчий НП у локальній базі (радіус 25 км), інакше — онлайн-сервіс. Якщо найближчий НП далі за 25 км, з'явиться попередження.
- Серійні номери, набрані кирилицею (Е, В, Р, К, А…), розпізнаються коректно.

## Поради

- Тримайте `Подання.html` у надійному місці, зробіть копію. Уся програма — в одному файлі.
- Дані зберігаються лише у вашому браузері на цьому комп'ютері. На іншому пристрої поля «Підрозділ/В.Ч.» доведеться заповнити заново, а реєстр — підключити ще раз.
